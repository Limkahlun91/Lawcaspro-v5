Module Development Prompt



When creating a new system module, always follow this structure.



Each module must include:



1\. Database schema / migration

2\. Backend service layer

3\. API routes

4\. Types / interfaces

5\. Authentication and permission checks

6\. Frontend UI components

7\. Error handling

8\. Logging

9\. Integration testing



Implementation must follow the system workflow:



1\. Analyze system architecture

2\. Identify affected modules

3\. Update database if required

4\. Implement backend services

5\. Create API routes

6\. Update types/interfaces

7\. Implement frontend components

8\. Ensure full system integration



Modules must support SaaS multi-tenant architecture.



Every record must be associated with firm\_id.



Never implement partial modules.
